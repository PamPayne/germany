## 3 Application for the test

The personal data provided above may be used for a transliteration test. To do this, a player must be created with this personal data, which must then be created in the LUGAS test system, see “Technical guidelines for central files”, section “Create a player”. When creating the player, it must be noted that the limitAmount field is not set. This ensures that no new player is registered, but that the already registered player should be addressed with the exemplary personal data. The call to create a player if the limitAmount is not set will only be successful if the player is found in the LUGAS test system, i.e. is already registered.

Please note that the personal data and the limit of the player registered with the personal data specified above may not be changed. For this player, only creation without a set limitAmount and the request to set ACTIVE are permitted.

The player registered with the personal data provided above is permanently set to ACTIVE in the LUGAS test system and can therefore also be used to test a request to set an already active player to ACTIVE. A request to set ACTIVE will only be successful if the player in question is set INACTIVE. An error message in response to the request to set ACTIVE confirms that the player was addressed with the exemplary personal data.

A successful response therefore confirms the correct transliteration of the personal data.
